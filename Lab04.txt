Step 1: Clone Lab Instructions
------

Open the VSCode Terminal.
Navigate to the AWSLabs directory
Pull the lab repository:
	git pull https://github.com/awscloudessentials/intro-to-aws-lab.git

Open the file Lab03.txt in the editor.
Document the steps you take while completing the lab inside this file.

 
Step 2: Launch EC2 Instance and Create Snapshot
------

Note: Use the Key Pair and Security Group created in the previous lab.
Launch a new EC2 Instance.
SSH into the instance.
Create a text file:
	sudo touch ebs_backup.txt
   
Confirm the file exists:
	ls -la
   
Take a screenshot for confirmation.
Go to Elastic Block Store → Snapshots.
Click Create Snapshot and configure:
	- Resource Type: Instance
	- Select your Instance ID
	- Description: Snapshot from instance1
Use this as Tag:
	- Key: Name
	- Value: snapshot1
Click Create Snapshot.
Wait until the snapshot Status = Completed.
Once completed, Terminate the instance.


Step 3: Create a Volume from the Snapshot
------

Go to Elastic Block Store → Snapshots.
Select the snapshot you created.
Click Actions → Create Volume from Snapshot.
Note the Availability Zone (AZ).
Add the following Tag:
	- Key: Name
	- Value: Restored volume
Create the volume.
Delete the snapshot.
Go to Elastic Block Store → Volumes and confirm that the Restored Volume exists.


Step 4: Attach Restored Volume to New Instance
------

Note: Use the same Key Pair and Security Group from the previous lab.

Launch a new EC2 instance in the same Availability Zone as the restored volume.
Under Network Settings, ensure the correct AZ is selected.
Create the instance.
Stop the instance (do not terminate).
Note the volume attached to the instance.
Go to Elastic Block Store → Volumes.
Select the default volume attached to the instance.
Click Detach Volume.
Select the Restored Volume (status should be Available).
Click Actions → Attach Volume.
Configure:
	- Instance: Select your new instance
	- Device Name: change from /dev/sdf to /dev/xvda
Click Attach Volume.
Delete the original volume you detached.


Step 5: Verify the Restored File
------

Start the new EC2 instance.
SSH into the instance.
Run:
	ls -la
   
Confirm that the file ebs_backup.txt is present.
Take a screenshot for confirmation.


Step 6: Cleanup and Submission
------

Clean up your AWS account resources:
	- Terminate instances
	- Delete volumes if necessary
Edit the Lab3 file in VSCode with the steps you completed.
Save the file.
Push it to your personal GitHub repository.
