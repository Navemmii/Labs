Open the Lab5 file in the editor.

Step 2: Create a NAT Gateway
------

Go to the AWS Management Console.
Navigate to VPC Dashboard.
Select NAT Gateways.
Click Create NAT Gateway.
Configure the following:
	- Name: NAT
	- Subnet: Select a public subnet
	- Connectivity Type: Public
Click Allocate Elastic IP.
Click Create NAT Gateway.


Step 3: Create and Configure a Private Route Table
------

Go to VPC Dashboard → Route Tables.
Click Create Route Table.
Configure:
	- Name: privateRT
	- VPC: Select your Default VPC

Click Create Route Table.
Select the newly created route table.
Go to the Routes tab.
Click Edit Routes → Add Route.
Configure the route:
	- Destination: 0.0.0.0/0
	- Target: Select the NAT Gateway created in Step 2

Click Save Changes.
Modify Subnet Associations
Select the Default Route Table.
Go to Subnet Associations.
Click Edit Subnet Associations.
Unselect one subnet and note its Subnet ID. (This subnet will be used to launch the private EC2 instance.)
Click Save Associations.
Select the privateRT route table.
Go to Subnet Associations → Edit Subnet Associations.
Select the subnet removed from the default route table.
Click Save Associations.


Step 4: Launch Public and Private EC2 Instances
------

Use the same key pair for both instances.

Public EC2 Instance
Create an EC2 instance with the following:
	- Name: public-ec2
	- Public IP: Enabled
	- Internet Access: Enabled
	- SSH Access: Enabled

Private EC2 Instance
Create another EC2 instance with the following configuration:
	- Name: private-ec2
	- Subnet: Select the subnet configured in Step 3
	- Auto-assign Public IP: Disabled
Security Group configuration:
	- Allow All IPv4 traffic from:
		- Source: Private IP of the public instance
	- Allow SSH from All sources


Step 5: Set Up SSH Agent Forwarding
------

Open WSL terminal.
Navigate to the location of your key pair.
Move the key to the SSH directory:
	mv <keypairname> ~/.ssh/<keyname>

Start the SSH agent:
	eval $(ssh-agent)

Set correct permissions:
	chmod 400 ~/.ssh/<keyname>

Add the key to the SSH agent:
	ssh-add -k ~/.ssh/<keyname>

Connect to the public EC2 instance:
	ssh -A -i ~/.ssh/<keyname> ec2-user@<public-ip-of-public-ec2>


Step 6: Verify Connectivity
------

Run the following commands and take screenshots for verification.
Test Internet Connectivity
	ping google.com

Test Connectivity to Private Instance
	ping <private-ip-of-private-ec2>

SSH into Private Instance
	ssh ec2-user@<private-ip-of-private-ec2>

Verify that the IP address is different, confirming you are inside the private instance.
Verify Internet Access from Private Instance
	ping google.com

This should work because the NAT Gateway allows outbound internet access.
Take screenshots for each verification step.
