Step 1: Clone Lab Instructions
------

Open the VSCode Terminal.
Navigate to the AWSLabs directory
Pull the lab repository:
	git pull https://github.com/awscloudessentials/intro-to-aws-lab.git
   
Open the Lab09 file in the editor.
Use this file to document the steps you take to complete the lab.


Step 2: Create an S3 Bucket
------

Go to the S3 Dashboard on the AWS Console.
Click Create Bucket.
Configure the following:
	- Bucket Name: Choose a unique name.
	- Region: Select any region of your preference.

Under Object Ownership, select:
	- ACLs enabled

Under Block Public Access settings:
	- Unselect Block all public access.
	- Check the acknowledgement box.

Click Create Bucket.


Step 3: Upload an Object
------

Open the bucket you created.
Click Upload → Add Files.
Select a good picture of yourself from your local machine.
Scroll down to Permissions.
Select:
	- Grant public-read access

Check the acknowledgement box.
Click Upload.


Step 4: Access the Uploaded Object
------

Open the uploaded object in the bucket.
Copy the Object URL.
Paste the Object URL into a new browser tab.
Post the Object URL in the group chat for confirmation.
After confirmation, go to the Permissions tab of the object.
Under Block Public Access (Bucket Settings):
	- Click Edit
 	- Select Block all public access

Click Save Changes and confirm.


Step 5: Test Public Access Restriction
------

Try accessing the Object URL again in your browser.
Take a screenshot of the message displayed in the browser.
From the object dashboard, click Open.
Answer the following:
	- Were you still able to access the object?
	- What do you think is the difference between accessing via the Object URL and the Open button in the console?


Step 6: Restore Access Using Bucket Policy
------

Go back to the Permissions tab of the bucket.
Under Block Public Access:
	- Click Edit
 	- Unselect Block Public Access

Save the changes.
Verify that you can access the object again using the Object URL.

---
{Disable ACLs}
Scroll to Access Control List (ACL) settings.
Select:
	- ACLs Disabled

Try accessing the Object URL again.

---
{Add a Bucket Policy}
Scroll to Bucket Policy.
Click Edit.
Paste the following JSON policy.
Note: Replace DOC-EXAMPLE-BUCKET with your actual bucket name.

json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicRead",
            "Effect": "Allow",
            "Principal": "",
            "Action": [
                "s3:GetObject",
                "s3:GetObjectVersion"
            ],
            "Resource": [
                "arn:aws:s3:::DOC-EXAMPLE-BUCKET/"
            ]
        }
    ]
}


Click Save Changes.
Verify that you can access the object again via the Object URL.
