Step 2: Create a Launch Template
------

Log in to your AWS Management Console.
Go to EC2 Dashboard → Launch Templates.
Click Create Launch Template.
Configure the following:
- Basic Settings
	Template Name: Any name (e.g., ASG-LT)
	VPC: Default VPC
- AMI and Instance Type
	AMI: Amazon Linux 2023
 	Instance Type: t3.micro
- Key Pair
	Key Pair Name: Linux
	Key Format: .pem
 	Download and save it to your local machine.
- Security Group
	Name: PublicSG
	Description: Any description
	Add inbound rules:

	| Type | Source    |
	| ---- | --------- |
	| SSH  | 0.0.0.0/0 |
	| HTTP | 0.0.0.0/0 |

- Network Settings
	Enable Auto-assign Public IP
- User Data Script

Paste the following script into the User Data section:

Step 3: Create Auto Scaling Group (ASG)
------

Go to EC2 Dashboard → Auto Scaling Groups.
Click Create Auto Scaling Group.
- Configure:
	Name: ASG
	Launch Template: Select the template created in Step 2
	VPC: Default VPC
	Subnets: Select available subnets
- Scaling Configuration
	Minimum Capacity: 1  
	Desired Capacity: 2  
	Maximum Capacity: 3  
- Scaling Policy: None
Click Next → Create Auto Scaling Group


Step 4: Verify Auto Scaling Behavior
------

Go to EC2 → Instances.
Wait until 2 instances are running.
Select the running instances
Click Instance State → Terminate
Observe:
	The instances will terminate
	The Auto Scaling Group will automatically launch new instances
Take a screenshot showing:
	Old instances terminating
	New instances launching
Send for verification.
