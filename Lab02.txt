Open your VSCode terminal and run:

Then open Lab02.txt in VSCode to read any additional instructions included there.

Step 2 — Create a Key Pair

In the AWS Console, search for EC2 and open it
In the left sidebar, scroll to Network & Security → Key Pairs
Click Create key pair
Fill in:
   - Name: Linux
   - Key pair type: RSA
   - Private key file format: .pem
5. Click Create key pair — the .pem file will auto-download to your machine

Step 3 — Create a Security Group

In EC2 left sidebar → Network & Security → Security Groups
Click Create security group
Fill in:
   - Name: PublicSG
   - Description: e.g. "Public security group for web server"
   - VPC: Select your default VPC
Under Inbound rules, add three rules:
| Type | Protocol | Port | Source |
|------|----------|------|--------|
| SSH  | TCP      | 22   | My IP  |
| HTTP | TCP      | 80   | 0.0.0.0/0 (Anywhere IPv4) |
| HTTP | TCP      | 80   | ::/0 (Anywhere IPv6) |

Leave outbound rules as default, then click Create security group

Step 4 — Create a Launch Template

In EC2 left sidebar → Instances → Launch Templates
Click Create launch template
Fill Name & description: anything you like (e.g. MyWebServerTemplate)
Application and OS Images: Click Quick Start → Amazon Linux → select Amazon Linux 2023 AMI (kernel 6.1)
Instance type: t3.micro
Key pair: Select Linux (the one you created in Step 2)
Security groups: Choose Select existing → pick PublicSG
Scroll down to Advanced details → scroll to the bottom to find User data
Paste this script:

#!/bin/bash
# Please make sure to launch Amazon Linux 2023 kernel 6.1 AMI
dnf update -y
dnf install -y httpd
systemctl start httpd
systemctl enable httpd
echo "Hello World from $(hostname -f)" > /var/www/html/index.html
echo "Healthy" > /var/www/html/health.html

Click Create launch template
