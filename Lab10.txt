Installing and Configuring AWS CLI

Step 1: Clone Lab Instructions
------

Open the VSCode Terminal.
Navigate to the AWSLabs directory
Pull the lab repository:
	git pull https://github.com/awscloudessentials/intro-to-aws-lab.git

Open the file Lab10.txt in the editor.
Document the steps you take while completing the lab inside this file.

Step 2: Create AWS Programmatic Access Credentials
------

(Ignore this step if you already have AWS programmatic credentials.)

Click on your account name at the top right corner of the AWS Console.
Select Security Credentials.
Scroll down to Access Keys.
Click Create Access Key.
Download the generated .csv file.
Save the file securely on your local system.


Step 3: Install AWS CLI
------

(Ignore this step if AWS CLI is already installed.)
Open a WSL terminal in VSCode.
Run the following commands:
Download AWS CLI:
	curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"

Install unzip (if not already installed):
	sudo apt install unzip

Extract the installer:
	unzip awscliv2.zip

Run the installation:
	sudo ./aws/install

Verify installation:
	aws --version

Reference:
[https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)


Step 4: Configure AWS CLI
------

Set up your default AWS CLI profile by running:
	aws configure

Enter the following information:
	- AWS Access Key ID → From the `.csv` file downloaded in Step 2
 	- AWS Secret Access Key → From the `.csv` file
 	- Default region name → `us-east-1` (or your preferred region)
	- Default output format → `json` or `yaml`

Reference:
[https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-files.html](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-files.html)


Step 5: Verify AWS CLI Configuration
------

Run the following command to list your S3 buckets:
	aws s3 ls

Go to the AWS S3 Dashboard in the console.
Create a new bucket.
Run the command again:
	aws s3 ls

Verify that the newly created bucket appears in the terminal output.
Take a screenshot for confirmation.


Step 6: Cleanup and Submission
------

Clean up your AWS resources:
	- Delete the S3 bucket you created.

Return to VSCode.
Edit the Lab10 file and document all the steps you performed.
Save the file.
